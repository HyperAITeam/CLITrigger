## 2026-05-30 — 사용자용 "세션" 명칭을 "터미널"로 통일 + 탭별 터미널 새로고침 버튼

### 배경

내부적으로는 "session"이라 부르던 개념이 사용자 입장에서는 사실상 하나의 터미널 창이므로, UI 문구를 "터미널"로 통일했다. 또한 xterm.js 뷰가 가끔 깨지거나(리사이즈 잔상, 스크롤백 꼬임) 다시 그리고 싶을 때가 있는데, PTY를 죽이지 않고 화면만 재구성할 수단이 없어 탭별 새로고침 버튼을 추가했다.

### 주요 변경

#### 1. 사용자용 "session" → "terminal" 문구 통일 (`eb2b519`)

- **클라이언트**: 설정 탭, 패널 제목, 힌트, 태그, 확인 다이얼로그, 세션 폼/리스트 라벨, 탭, 팝아웃 재할당 안내 문구를 모두 "Terminal"(EN) / "터미널"(KO)로 변경.
- **i18n**: `src/client/src/i18n.tsx`의 해당 라벨 문자열만 교체.
- **유지**: DB 테이블, 라우트(`/api/.../session*`), i18n **키** 이름은 `session` 그대로 둠 — UI 표시 문자열만 변경. (Memory→Wiki 리네이밍과 같은 원칙: 내부 식별자는 보존.)

#### 2. 탭별 터미널 새로고침 버튼 (`b9d6e41`)

- **`StackView.tsx`**: 툴바에 새로고침 버튼(`RotateCw`) 추가. 탭별 remount 카운터를 증가시켜 활성 탭의 `SessionTerminal`을 강제 재마운트.
- **`SessionPane.tsx`**: `remountKey` prop을 받아 `SessionTerminal`의 React key에 섞음 → xterm.js 인스턴스를 dispose/재생성. **PTY는 살아 있고** 재구독 시 DB 청크에서 출력이 그대로 replay되므로 화면만 새로 그려짐.
- **i18n**: `session.refresh` / `session.refresh.hint` 한·영 추가.

### 아키텍처 결정

1. **표시 문자열만 리네이밍, 내부 식별자는 보존**: DB 테이블/라우트/i18n 키의 `session` 명칭은 유지해 마이그레이션·API 호환성을 깨지 않음. UI 용어와 코드 식별자를 분리.
2. **새로고침은 xterm 재마운트, PTY 재시작 아님**: DB `session_raw_chunks`가 replay의 단일 소스이므로 xterm.js만 dispose/재생성하면 재구독으로 화면이 동일하게 복원됨. PTY 프로세스를 건드리지 않아 실행 중인 작업이 끊기지 않음.
