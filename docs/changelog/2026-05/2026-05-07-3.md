## 2026-05-07 — 빌드 스크립트 PFX 패스워드 환경변수 분리

### 배경

`build-win.bat`에 MSIX 자가서명 인증서 패스워드 `clitrigger`가 평문으로 하드코딩되어 있어 GitGuardian이 GitHub push 후 시크릿 노출 경보를 발송했다. 실질적 위험은 없는 로컬 테스트용 인증서이지만 스캐너 alert를 없애고 사용자가 원하는 패스워드를 외부에서 주입할 수 있도록 개선했다.

### 주요 변경

#### 1. CSC_KEY_PASSWORD 환경변수 fallback 도입 (`e088cdb`)

- **`build-win.bat`**: 스크립트 상단에 `if not defined CSC_KEY_PASSWORD set CSC_KEY_PASSWORD=clitrigger` fallback 추가. 인증서 생성(`ConvertTo-SecureString`), electron-builder 전달(`CSC_KEY_PASSWORD` env), 설치 안내 메시지(`Import-PfxCertificate`) 세 곳 모두 `%CSC_KEY_PASSWORD%` 참조로 교체.
- 환경변수 미설정 시 기존 동작(기본값 `clitrigger`) 유지. 설정 시: `set CSC_KEY_PASSWORD=원하는패스워드 && build-win.bat --msix`

### 수정된 주요 파일

| 파일 | 변경 내용 |
|------|----------|
| `build-win.bat` | 하드코딩 패스워드 → `%CSC_KEY_PASSWORD%` 참조 (fallback 기본값 유지) |
