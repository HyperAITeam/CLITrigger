## 2026-06-10 — 터미널 이미지 붙여넣기 썸네일 미리보기

### 배경

세션 터미널에 이미지를 붙여넣으면 CLI 화면에는 `[Image #N]` 토큰만 표시되어, 방금 무엇을 붙여넣었는지 시각적으로 확인할 방법이 없었다. 한글 IME 컴포지션 미러(좌하단 오버레이)와 같은 방식의 미리보기가 필요했다.

### 주요 변경

#### 1. 좌하단 썸네일 오버레이 (`7ee7861`)

- **클라이언트**: 이미지 붙여넣기 시 IME 미러와 같은 좌하단 위치에 썸네일(최대 220×140) + 용량(KB/MB)을 3초간 표시 후 자동 소멸. 미리보기 중 새 이미지를 붙여넣으면 타이머가 리셋되며 교체된다.
- 이미지 붙여넣기 두 경로 모두 연결 — `clipboard.read()` 경로(보안 컨텍스트 Ctrl+V)와 네이티브 `paste` 이벤트 폴백(HTTP/LAN 접속, 우클릭 메뉴). 둘 다 이미 dataURL을 만들고 있어 서버 업로드 플로우는 변경 없음.
- IME 컴포지션 박스와 동시 표시될 때 겹치지 않도록 좌하단을 flex column 스택으로 구성. 둘 다 `pointer-events: none`이라 터미널 클릭/선택을 방해하지 않는다.
- 클립보드에 이미지 데이터가 아닌 **파일 경로**만 있는 경우(`getClipboardImagePath` 폴백)는 클라이언트에 이미지 데이터가 없어 미리보기 대상에서 제외.

### 수정된 주요 파일

| 파일 | 변경 내용 |
|------|----------|
| `src/client/src/components/SessionTerminal.tsx` | `pastedImage` 상태 + 3초 자동 소멸 타이머, 두 붙여넣기 경로에 미리보기 호출, 좌하단 오버레이 스택 렌더링 |

### 아키텍처 결정

1. **dataURL 재사용**: 두 붙여넣기 경로 모두 서버 업로드용 dataURL을 이미 생성하므로, 그것을 그대로 `<img src>`에 재사용해 추가 디코딩/요청 없이 미리보기를 구현했다.
