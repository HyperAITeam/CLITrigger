## 2026-06-11 — AI 모델 선택 기능 전면 제거: 항상 CLI 기본 모델로 실행

### 배경

모델 라인업이 계속 바뀌면서 프로젝트·TODO·세션·스케줄·에이전트마다 흩어진 모델 드롭다운 목록을 따라 업데이트하는 것이 비현실적이 됐다. 모델 선택을 제품에서 제거하고 실행은 무조건 각 CLI의 default 모델을 쓰기로 했다.

### 주요 변경

#### 1. 실행 시 모델 무시 강제 (`49f3a4f`)

- **서버**: orchestrator(자동 작업) / session-manager(세션) / discussion-orchestrator(토론 에이전트)의 모델 결정 지점을 `undefined`로 고정. CLI 인자 빌더는 모델이 없으면 `--model`을 붙이지 않으므로, **DB에 남은 `cli_model`/`claude_model` 레거시 값도 실행 시 전부 무시**된다. 스케줄 발화 시 TODO로 모델이 복사되던 경로도 차단.
- 라우트(todos/sessions/schedules/planner/discussions/projects)에서 모델 필드 수신 제거 — 새 값이 저장될 수 없다. DB 컬럼·`/api/models`·model-sync는 마이그레이션 없이 유지(죽은 코드).

#### 2. 모델 UI 제거 (`49f3a4f`)

- **클라이언트**: 모델 드롭다운 제거 — 프로젝트 설정, TODO 폼, 세션 폼, 스케줄 폼, 플래너 변환 다이얼로그, 토론 에이전트 폼. TODO/태스크 그래프/세션 목록의 모델 배지 표시도 제거. **CLI 도구(Claude/Gemini/Codex) 선택은 유지.**
- `ModelSettings.tsx`(커스텀 모델 등록 화면)·`useModels` 훅·`api/models.ts` 삭제, `cli-tools.ts`를 정적 도구 목록으로 단순화. deprecated 모델 경고 배지도 함께 제거.
- **i18n**: `*.aiModel`·`header.model*`·`plannerConvert.model` 등 모델 관련 키 일괄 삭제.

#### 3. 설정 탭 이름 변경 (`49f3a4f`)

- 프로젝트 설정의 "실행 설정" 탭을 **"자동화 설정"(Automation Settings)**으로 변경 — 실제로 자동 작업·토론·스케줄 실행에만 적용되는 설정이므로.

### 수정된 주요 파일

| 파일 | 변경 내용 |
|------|----------|
| `src/server/services/orchestrator.ts` · `session-manager.ts` · `discussion-orchestrator.ts` | 모델 결정 지점 `undefined` 고정 |
| `src/server/routes/{todos,sessions,schedules,planner,discussions,projects}.ts` | 모델 필드 수신 제거 |
| `src/client/src/components/ProjectHeader.tsx` 외 폼 6종 | 모델 드롭다운/배지 제거, 탭명 변경 |
| `src/client/src/cli-tools.ts` | 모델 캐시/레지스트리 제거, 정적 도구 목록만 유지 |
| (삭제) `ModelSettings.tsx` · `hooks/useModels.ts` · `api/models.ts` | 고아 코드 정리 |

### 아키텍처 결정

1. **실행 지점에서 무시 (DB 마이그레이션 없음)**: 컬럼 삭제 대신 실행 경로 3곳에서 `undefined`를 강제했다. 기존 데이터에 안전하고, 라우트 수신 차단과 합쳐 "새 값도 못 들어오고 옛 값도 무시되는" 상태가 된다.
2. **CLI 도구 선택은 존치**: 도구(Claude/Gemini/Codex)는 안정적인 선택지이고 폴백 체인 등과 얽혀 있어 유지. 변동성이 큰 것은 모델 목록뿐이었다.
