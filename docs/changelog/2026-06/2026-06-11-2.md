## 2026-06-11 — 프로젝트 설정 정리: 워크스페이스 탭 제거 + 위키 자동 인제스트 폐지 + SVN 별도 탭

### 배경

모델 제거 후 워크스페이스 탭에 남은 것은 위키 자동 수집 토글과 SVN 토글 두 개뿐이었다. 위키 자동 수집은 볼트(파일 기반 위키)로 대체된 구 기능이고, SVN 토글은 git 저장소에서는 아예 숨겨져 있었다(폴더가 git+svn 겸용일 수 있는데도).

### 주요 변경

#### 1. 위키 자동 인제스트 폐지 (`3581254`)

- **서버**: 작업 완료(orchestrator) / 토론 종료(discussion-orchestrator) 시 구 위키로 자동 인제스트하던 훅 삭제. 라우트에서 `memory_auto_ingest` 수신 제거 — 기존에 켜져 있던 프로젝트도 더 이상 자동 인제스트가 돌지 않는다.
- 위키 탭의 **수동** 인제스트 API와 DB 컬럼은 유지(위키 화면 자체는 존치).
- **클라이언트**: 토글 UI·payload·i18n 키 제거. 워크스페이스 탭 자체를 삭제.

#### 2. SVN 별도 설정 탭 + git 무관 노출 (`6886d55`)

- **서버**: `GET /api/projects/:id/svn-detect` 신설 — `svn_enabled`를 건드리지 않는 읽기 전용 워킹카피 감지(`.svn/` 확인 → `svn info` 폴백).
- **클라이언트**: 설정 패널을 열 때 감지를 호출해, **SVN 워킹카피이거나 이미 활성화된 프로젝트에서만** SVN 탭을 노출 — git 여부 무관. 설정 탭 구성은 `자동화 설정 / 하네스 / 플러그인 / SVN(조건부)`.
- **겸용 폴더 보호**: git 프로젝트는 SVN을 켜도 `vcs_type`을 'svn'으로 강등하지 않음 — `vcs_type='svn'`은 최대 워커 수를 1로 제한하는 정책에 쓰이는데, git 프로젝트는 worktree 병렬 실행을 보존해야 한다. SVN 탭/라우트는 `svn_enabled` 기준이라 영향 없음.

### 수정된 주요 파일

| 파일 | 변경 내용 |
|------|----------|
| `src/server/services/orchestrator.ts` · `discussion-orchestrator.ts` | 자동 인제스트 훅 제거 |
| `src/server/routes/projects.ts` | `memory_auto_ingest` 수신 제거, `svn-detect` 라우트, vcs_type 강등 방지 |
| `src/client/src/components/ProjectHeader.tsx` | 워크스페이스 탭 삭제, SVN 전용 탭(감지 기반 노출) |
| `src/client/src/api/projects.ts` | `detectSvn()` 추가, `memory_auto_ingest` 타입 제거 |

### 아키텍처 결정

1. **감지 기반 탭 노출**: SVN 탭을 모든 프로젝트에 보여주는 대신 설정 열 때 1회 감지로 가린다. "SVN 저장소면 보인다"는 직관과 일치하고, 비 SVN 프로젝트의 설정이 깔끔해진다.
2. **vcs_type은 git 우선**: 듀얼 VCS 폴더에서 vcs_type은 실행 정책(워커 수)을 결정하므로 더 능력 있는 쪽(git)을 유지하고, SVN UI는 별도 플래그(svn_enabled)로 구동한다.
