## 2026-07-01 — Gemini CLI를 Antigravity CLI(agy)로 교체

### 배경

Google이 2026-06-18부로 개인 계정(무료/Pro/Ultra, OAuth 로그인)용 Gemini CLI를 종료하고 후속 도구인 **Antigravity CLI(`agy`)** 로 대체했다. Gemini CLI로 실행되던 자동화 작업이 개인 로그인 사용자에게서 이미 중단된 상태라, CLI 프로바이더 `gemini`를 `antigravity`로 통째로 교체한다. (엔터프라이즈 Code Assist / API 키 사용자는 Gemini가 여전히 동작하지만, 제품 표준은 후속 도구로 이전한다.)

### 주요 변경

#### 1. CLI 프로바이더 교체 (`d6a5da0`)

- **타입 유니온(SoT)**: `CliId` / `CliTool`의 `'gemini'` → `'antigravity'` (서버·클라이언트 4곳). 모든 spawn·UI·DB 참조가 이 유니온을 단일 출처로 사용.
- **실행 어댑터**: `geminiAdapter` → `antigravityAdapter`. `command: 'agy'`, 비대화 모드는 `--headless`, 자동 승인은 `--dangerously-skip-permissions`, 세션 이어가기는 `--continue`. `probeModels()`는 `probeViaHelp('agy')`. 프롬프트는 기존과 동일하게 stdin 파이프로 전달.
- **하네스 어댑터**: `adapters/gemini.ts` → `adapters/antigravity.ts`. 컨텍스트 파일 `GEMINI.md` → `AGENTS.md`, 프로젝트 설정 `.gemini/settings.json` → `.agents/settings.json`, **MCP 서버는 settings.json 인라인이 아니라 별도 `.agents/mcp_config.json`** 로 분리(Antigravity의 변경점).
- **DB**: `seedCliModels`의 기본 모델을 `antigravity`로 교체 + `gemini`→`antigravity` idempotent 마이그레이션 추가. `cli_models` / `cli_versions` / `projects` / `todos` / `schedules` / `sessions` / `discussion_agents`의 `cli_tool` 값과 `projects.cli_fallback_chain` 내 토큰까지 재작성.
- **부가 실행 경로**: 설치 감지(`cli-status`), 토론·메모리 인제스트 spawn 명령(`discussion-extractor`, `memory-ingest`), 모델 레지스트리(`cli-models-registry.json`, `model-sync`)를 `agy`/`antigravity`로 갱신.
- **UI/i18n**: 설치 힌트(`curl -fsSL https://antigravity.google/cli/install.sh | bash`), 하네스 라벨·설명, `SettingsForm` 승인/샌드박스 옵션, 플래너 변환 드롭다운, gstack 안내 문구.
- **테스트**: 어댑터·하네스·model-sync·queries 테스트를 Antigravity로 갱신(서버 172개 통과).

> 참고: `agy` 미설치 상태에서 문서 기반으로 구현했으며, submit 시퀀스·ready 패턴·auto-respond·승인/샌드박스 옵션·프로젝트 설정 경로/MCP 포맷 등 일부 값은 코드에 `PROVISIONAL` 주석으로 표시. 실제 `agy` 설치 후 `agy --help` / `agy inspect`로 실측 보정 필요.

#### 2. 사용자 노출 텍스트 및 주석 정리 (`5457210`, `dd1bc25`)

- 작업/세션 탭 도움말 텍스트의 CLI 나열을 `Claude/Gemini/Codex` → `Claude/Antigravity/Codex`로(en·ko).
- 현재 지원 CLI를 잘못 가리키던 코드 주석 다수를 Antigravity/`agy`로 갱신. 역사적 참조("구 Gemini CLI와의 차이"), 다중 벤더 모델 매칭 정규식, DB 마이그레이션 코드의 `gemini` 언급은 의도적으로 유지.

### 아키텍처 결정

1. **새 식별자 + DB 마이그레이션**: 내부 id를 `'gemini'`로 유지하지 않고 `'antigravity'`로 교체. 바이너리·모델·설정 스키마·컨텍스트 파일이 전부 달라 id를 유지하면 영구 오칭이 되므로, 기존 `schema.ts`의 idempotent try/catch 패턴을 따르는 저비용 마이그레이션으로 기존 데이터를 재작성.
2. **MCP 설정 파일 분리**: Antigravity는 MCP 서버를 settings.json 인라인이 아니라 전용 `mcp_config.json`으로 관리하므로, 하네스 어댑터의 read/upsert/remove를 두 파일로 분리.
3. **모델 ID는 런타임 probe에 위임**: `agy`의 정확한 모델 식별자를 문서로 확정할 수 없어 레지스트리는 비워두고, 시드된 Default(빈 model_value = CLI 기본) + `probeViaHelp('agy')` 실측으로 채우도록 구성.
