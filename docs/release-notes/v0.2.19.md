# v0.2.19 — 터미널 도킹 대확장 · 모델 선택 제거 · 볼트 온보딩

릴리즈 일자: 2026-06-11
이전 버전: v0.2.18

## 요약 (TL;DR)

별도 OS 창으로 빼낸 터미널(팝아웃)이 이제 진짜 도킹을 합니다 — 팝아웃 안에서 스플릿을 만들고, 창과 창 사이로 탭을 끌어 옮기고, 서로 다른 프로젝트의 터미널을 한 그룹으로 묶을 수 있습니다. 터미널에 이미지를 붙여넣으면 썸네일 미리보기가 떠서 뭘 첨부했는지 바로 보입니다. AI 모델 선택 UI는 전부 제거되어 항상 각 CLI의 기본 모델로 실행되고, 큰 프로젝트의 볼트 첫 진입 렉은 "전부 숨김으로 시작" 온보딩으로 해결했습니다.

## 주요 변경

### 세션 윈도우 도킹 확장

- **팝아웃 내부 도킹**: 별도 OS 창 안에서도 탭을 드래그하면 메인 창과 같은 5-zone 독 오버레이가 뜹니다. 자기 스택의 가장자리에 드롭해 그 자리에서 스플릿을 만들 수 있습니다.
- **OS 창 간 탭 드래그**: 팝아웃의 탭을 다른 팝아웃 창이나 메인 창의 플로팅 터미널 위로 끌어다 놓으면 탭이 그 창으로 이동합니다. 창이 겹치면 가장 최근에 포커스된 창이 받고, 수신 창이 확인한 뒤에만 탭이 빠지므로 도중에 세션이 사라질 일이 없습니다.
- **크로스 프로젝트 도킹**: 다른 프로젝트를 보는 중에 팝아웃을 Re-dock하면 지금 보고 있는 워크스페이스로 들어옵니다. 이 경로로 서로 다른 프로젝트의 터미널을 한 그룹(스플릿/탭)으로 묶을 수 있습니다.

### 터미널 이미지 붙여넣기 미리보기

- 이미지를 붙여넣으면 CLI에 `[Image #N]`만 보이던 것에서, 터미널 좌하단에 썸네일 + 용량이 3초간 표시되어 무엇이 첨부됐는지 즉시 확인할 수 있습니다.

### 볼트 첫 진입 온보딩 (큰 프로젝트 렉 해결)

- 볼트 탭 첫 방문 시 스캔/그래프 렌더링을 시작하기 전에 온보딩 다이얼로그가 뜹니다. **"전부 숨김으로 시작"**(권장)을 고르면 `.vaultignore`에 `*`가 생성되어 가볍게 출발하고, 파일 탐색기에서 회색 파일을 우클릭 → "볼트에 다시 보이기"로 필요한 문서만 노출합니다.
- "다시 보이기"가 gitignore negation을 자동 생성하도록 재구현되어, 전부 숨김 상태에서도 개별 파일/폴더 해제가 정확히 동작합니다.

### 프로젝트 설정 정리

- "실행 설정" 탭이 **"자동화 설정"**으로 이름이 바뀌었습니다 (자동 작업·토론·스케줄에만 적용되는 설정이므로).
- 워크스페이스 탭이 사라지고, **SVN은 전용 설정 탭**으로 분리되었습니다 — 폴더가 SVN 워킹카피면 git 저장소여도 탭이 보입니다(듀얼 VCS). git 프로젝트는 SVN을 켜도 worktree 병렬 실행이 유지됩니다.

### 하네스 Claude 탭 개편

- 메모리 파일 에디터가 실제 파일명(`CLAUDE.md` 등)으로 표시되고, 확대 토글로 긴 파일도 잘리지 않게 편집할 수 있습니다.
- **CLAUDE.local.md**(없으면 생성 버튼), **Hooks**(이벤트별 목록 + JSON 편집), **Skills**(.claude/skills 목록 + 인라인 편집)가 추가되었습니다.

### 기타

- Windows 설치 시 UAC/설치 창에 긴 영어 설명문 대신 **CLITrigger**가 표시됩니다 (exe FileDescription 수정).

## ⚠️ Breaking Changes

- **AI 모델 선택 기능이 제거되었습니다.** 프로젝트 설정·TODO·세션·스케줄·플래너 변환·토론 에이전트의 모델 드롭다운과 "모델 관리" 화면이 모두 사라졌고, **모든 실행은 각 CLI의 기본(default) 모델을 사용합니다.** 기존에 저장돼 있던 모델 값도 실행 시 무시됩니다. CLI 도구(Claude/Gemini/Codex) 선택은 그대로입니다.
- **위키(구 Memory) 자동 인제스트가 폐지되었습니다.** 작업 완료/토론 종료 시 구 위키로 자동 수집하던 동작이 더 이상 돌지 않습니다 (수동 인제스트와 위키 화면 자체는 유지). 볼트(파일 기반 위키)가 후속 경로입니다.

## 마이그레이션

- 별도 조치는 필요 없습니다 — DB 마이그레이션 없이 기존 데이터가 그대로 동작합니다.
- 특정 모델을 강제하고 싶다면 각 CLI 자체 설정(예: `.claude/settings.json`의 `model` — 하네스 탭에서 편집 가능)을 사용하세요. CLITrigger는 더 이상 `--model` 플래그를 붙이지 않습니다.
