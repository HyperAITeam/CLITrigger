# v0.2.23 — Antigravity CLI(agy) 전환 + Windows IME 진단 로깅

릴리즈 일자: 2026-07-01
이전 버전: v0.2.22

## 요약 (TL;DR)

- 종료된 Gemini CLI를 후속 도구 **Antigravity CLI(`agy`)** 로 전면 교체했습니다. 기존 프로젝트/작업/스케줄에 남아 있던 `gemini` 설정은 업데이트 시 자동으로 `antigravity`로 마이그레이션됩니다.
- 패키징(exe) 환경에서 재현되는 한국어 IME 조합창 위치 버그를 실측하기 위한 **진단 로깅 토글**을 설정 > 터미널에 추가했습니다. 기본은 꺼짐이며, 필요할 때만 켜서 원인을 수집합니다.

## 주요 변경

### Gemini CLI → Antigravity CLI(agy) 교체

Google이 개인 계정용 Gemini CLI를 종료(2026-06-18)하고 후속 도구 Antigravity CLI로 대체함에 따라, CLI 프로바이더를 통째로 이전했습니다.

- CLI 선택 목록에서 **Gemini 대신 Antigravity**가 표시되며, 실행 바이너리는 `agy`입니다.
- 설치 안내가 Antigravity 기준(`curl -fsSL https://antigravity.google/cli/install.sh | bash`)으로 바뀌었습니다.
- 컨텍스트 파일이 `GEMINI.md` → `AGENTS.md`, 프로젝트 설정이 `.gemini/settings.json` → `.agents/settings.json`로 바뀌었고, MCP 서버 설정은 별도 `.agents/mcp_config.json`으로 분리됩니다.
- 작업/세션 탭 도움말, 승인/샌드박스 옵션, 플래너 변환 드롭다운 등 사용자 노출 텍스트를 모두 Antigravity로 갱신했습니다.

> 참고: `agy` 정식 미설치 상태에서 문서 기반으로 구현되어, 일부 옵션·모델 식별자는 실제 `agy` 설치 후 실측 보정이 필요할 수 있습니다(코드에 `PROVISIONAL` 표기). 엔터프라이즈 Code Assist / API 키 사용자는 Gemini가 여전히 동작하지만, 제품 표준은 후속 도구로 이전했습니다.

### Windows exe IME 진단 로깅 토글

패키징된 exe에서 한국어 IME 조합창이 화면 좌상단에 고정되는 증상(Chromium native-window-occlusion 오판)이 v0.2.20의 occlusion 비활성화 스위치 이후에도 재현되어, 잔여 원인을 실제 exe에서 측정하기 위한 계측을 추가했습니다.

- **설정 > 터미널**에 진단 로깅 토글을 추가했습니다. 재시작 후에도 유지되며(세션 설정에 저장), 기본은 꺼짐입니다.
- 켜면 조합 시작 시점의 상태와 커서 위치 계산 실패 사유가 `userData/ime-debug.log`에 기록되고, 창 focus/blur/show/hide 및 시작 스냅샷이 함께 남습니다.

## 마이그레이션

- 업데이트만 하면 됩니다. `cli_models`, `projects`, `todos`, `schedules`, `sessions` 등에 남아 있던 `gemini` 값과 `cli_fallback_chain` 토큰이 시작 시 **idempotent 마이그레이션**으로 `antigravity`로 자동 재작성됩니다.
- Antigravity 작업을 실제로 실행하려면 `agy` CLI 설치가 필요합니다.
